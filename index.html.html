1. Introduction — what is HTML?

HTML stands for HyperText Markup Language. It's the skeleton of every webpage. Think of HTML as the structure and content (headings, paragraphs, images), CSS as the clothing (style), and JavaScript as the behavior (actions).

2. Basic document structure

Every HTML file usually starts like this. Save it as index.html and open in your browser.

<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>My Page</title>
  </head>
  <body>
    <h1>Hello, world!</h1>
    <p>This is my first page.</p>
  </body>
</html>

3. Text, headings & paragraphs

Use headings for structure: <h1> is top-level, down to <h6>. Use <p> for paragraphs. Use <strong> and <em> for emphasis.

Example: 
<h1>Welcome</h1>
<p>This is a simple paragraph. You can <strong>bold</strong> or <em>italicize</em> text.</p>

4. Links & Images

Links use <a href="...">. Images use <img src="..." alt="...">. Always include alt for accessibility.

<a href="https://example.com" target="_blank" rel="noopener">Visit Example</a>

<img src="https://via.placeholder.com/200" alt="Placeholder image" width="200">

5. Lists & Tables

Use <ul> for unordered lists, <ol> for ordered. Tables use <table>, but keep tables for tabular data.

<ul>
  <li>Apples</li>
  <li>Bananas</li>
</ul>

<table>
  <thead>
    <tr><th>Name</th><th>Age</th></tr>
  </thead>
  <tbody>
    <tr><td>Aisha</td><td>28</td></tr>
  </tbody>
</table>

6. Forms (collecting user input)

Forms let you collect input. Each input needs a name. Use labels for usability and accessibility.

<form action="/submit" method="post">
  <label>Name: <input type="text" name="name" required></label>
  <label>Email: <input type="email" name="email" required></label>
  <button type="submit">Send</button>
</form>

7. Semantic HTML (meaningful tags)

Semantic tags help browsers and screen readers understand content: <header>, <nav>, <main>, <article>, <section>, <aside>, <footer>.

<header><h1>Blog</h1></header>
<main>
  <article>...</article>
  <aside>Related links</aside>
</main>
<footer>© 2025</footer>

8. Layout: Flexbox & Grid basics

Use CSS for layout. Here are tiny examples to start. You will write CSS inside <style> or an external file.

/* Flexbox */
.container{display:flex;gap:12px}
.item{flex:1;padding:12px}

/* Grid */
.grid{display:grid;grid-template-columns:1fr 2fr;gap:12px}

Flexbox is great for 1D (row/column). Grid is best for 2D layouts.

9. Audio & Video

Use the native tags. Provide captions or transcripts when possible.

<audio controls><source src="audio.mp3" type="audio/mpeg">Your browser doesn't support audio.</audio>

<video controls width="480"><source src="video.mp4" type="video/mp4">Your browser doesn't support video.</video>

10. Accessibility (a11y) tips

Small steps that make a big difference:

  • Use descriptive alt text for images.
  • Use <label> for form controls.
  • Provide keyboard focus styles and logical tab order.
  • Use ARIA only when necessary (don't replace semantic HTML).

11. Meta tags & SEO basics

Important meta tags go in the head:

<meta name="description" content="Short description of the page">
<meta name="viewport" content="width=device-width, initial-scale=1">

12. Quick cheat sheet

Common tags at a glance:

<!-- Document -->
<!doctype html> <html> <head> <body>

<!-- Text --> <h1> <h2> <p> <strong> <em>

<!-- Links & Images --> <a href> <img src alt>

<!-- Lists --> <ul> <ol> <li>

<!-- Forms --> <form> <input> <label> <button>

<!-- Semantic --> <header> <nav> <main> <article> <footer>

13. Practice challenges

Try these to learn faster:

  1. Create a personal profile page with a photo and contact form.
  2. Build a two-column layout using CSS Grid that becomes one column on mobile.
  3. Make an accessible navigation menu — keyboard friendly.
